### Особенности:  
- Из Mjpeg пакета парсится только картинка.
- Сам Mjpeg пакет - это статический класс, так как нужно хранить только картинку одного кадра.
- Сильно упрощённый поиск пакетов в стриме.
- События инициализируются пустым обработчиком, чтобы не проверять на null.
- Некоторые настройки, например адрес XML конфига, хранятся в App.config.

### Недостатки:
- Сильная связность между классами.
- Перевод картинки в стрим и обратно дважды, один раз в парсере, второй при отображении. Можно решить не парсив картинку, а оставив byte[].
- Приложение запускается только после загрузки XML конфига (загрузка названий каналов для выбора), иногда приходится ждать.
- ViewModel получилась достаточно сложной. Можно разделить ещё больше.
- Кнопка на время загрузки канала не отключается, а только игнорируется. Можно дописать Command класс и использовать CommandManager.InvalidateRequerySuggested, но это выглядело не очень.

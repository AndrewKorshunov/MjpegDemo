Особенности:
	- Особенный алгоритм прасинга пакетов.
	- События инициализируются пустым обработчиком, чтобы не проверять на null

Недостатки:
	- Перевод картинки в стри и обратно дважды, один раз в парсере, второй при отображении.
	- Приложение запускается только после загрузки XML конфига, иногда приходится долго ждать.
	- Если загружать канал, который в данный момент ожидает подключения, падает WebClient.
